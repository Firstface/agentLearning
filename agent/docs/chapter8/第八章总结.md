# 第八章 记忆与检索 — 总结文档

> 摘自 `docs/chapter8/第八章 记忆与检索.md` 的核心理念与实现要点。
> 不含代码，只讲"是怎么做的、关键在哪、有什么细节"。

---

## 一、本章想解决什么问题

LLM 本身有两个硬伤：

1. **无状态**：每次调用是独立的，会话之间天然遗忘，跨会话连贯性差。
2. **知识封闭**：训练数据有截止日，且不知道私有知识，回答容易过时或胡编乱造。

第八章提出两套互补方案：

| 方案 | 解决的问题 | 在 Agent 中的角色 |
|---|---|---|
| **记忆系统 (Memory)** | 让智能体"记得自己经历过什么、用户偏好什么、学到过什么概念" | 第一人称视角的"自传" |
| **RAG 系统 (Retrieval-Augmented Generation)** | 让智能体能"现查"私有/最新外部知识 | 第三人称视角的"参考书" |

设计灵感来自认知科学——把人类的多种记忆类型映射到工程实现上。

## 二、记忆系统的总体架构

### 1. 工具分层：MemoryTool → MemoryManager → 四种记忆模块 → 底层 Store

- **MemoryTool**：对外接口层，封装 `add / search / forget / consolidate / summary / stats / clear_all` 等动作。Agent 看到的就是这一层。
- **MemoryManager**：协调层，根据 `memory_type` 把请求路由到具体的记忆模块；不直接干活。
- **四种记忆模块**：每种记忆有自己的存储和评分策略。
- **底层 Store**：内存字典 / SQLite / Qdrant / Neo4j 等持久层。

这种分层的关键意义是 **关注点分离**：用户改不动底层、模块互相独立、可以按开关启用部分记忆类型。

### 2. 四种记忆类型对照（核心知识点）

| 类型 | 心理学对应 | 存什么 | 持久化 | 检索方式 |
|---|---|---|---|---|
| **WorkingMemory** | 工作记忆 | 当前会话临时片段 | 内存 + TTL 自动过期 | TF-IDF + 关键词混合 |
| **EpisodicMemory** | 情景记忆 | 具体事件（带时间地点上下文） | SQLite（结构化）+ Qdrant（向量）| 结构化过滤 + 向量语义 |
| **SemanticMemory** | 语义记忆 | 抽象概念、规则、知识 | Neo4j（图）+ Qdrant（向量）| 向量入口 + 图推理 |
| **PerceptualMemory** | 感知记忆 | 文本/图像/音频等多模态 | 多模态分集合的 Qdrant | 同模态向量检索（CLIP/CLAP）|

> 关键认知：**四种类型不是凭空设计的，而是为了解决不同的检索/生命周期需求**。
> - 工作记忆要快、能自动过期；情景记忆要按时间筛；语义记忆要做关系推理；感知记忆要支持多模态。
> - 一种存储引擎覆盖不了所有场景，所以分而治之。

### 3. 关键操作与机制

- **add（添加）**：补全 session_id、timestamp 等 metadata；对 perceptual 自动从文件后缀推断 modality 并把 raw_data 写入 metadata（用 `setdefault` 不覆盖用户已传值）。
- **search（检索）**：每个记忆类型有自己的"打分公式"，最终都是 `相关性 + 时间近因 + 重要性` 三项的加权乘积。
- **forget（遗忘）**：三种策略——基于重要性、基于时间、基于容量。模拟人类的遗忘机制，避免无限堆积。
- **consolidate（整合）**：把"重要工作记忆"提升到情景记忆，把"重复的情景记忆"抽象成语义记忆。这是从"经历"到"知识"的跃迁，对应人类的"记忆固化"。
- **summary / stats**：给 Agent 提供整体盘点能力。

### 4. 四个记忆类型的实现细节

#### 工作记忆（WorkingMemory）
- **特点**：纯内存、容量有限（默认 50 条）、TTL（默认 60 分钟）自动清理。
- **检索**：先尝试 TF-IDF 向量化，失败回退关键词匹配。
- **评分公式**：`(向量×0.7 + 关键词×0.3) × 时间衰减 × (0.8 + 重要性×0.4)`
- **关键设计**：靠 `_expire_old_memories` 在每次 add/retrieve 前清理过期项；容量满了就删"最低优先级"的一条。

#### 情景记忆（EpisodicMemory）
- **核心**：SQLite + Qdrant 双写。
  - SQLite 管"账本"——结构化字段（id、session_id、timestamp、importance、content）；
  - Qdrant 管"语义指纹"——内容 embedding。
- **检索**：先用 SQLite 按时间/重要性结构化过滤拿候选 ID；再用 Qdrant 向量找语义最相似的；最后取交集打分。
- **评分公式**：`(向量×0.8 + 时间近因×0.2) × (0.8 + 重要性×0.4)`
- **`self.sessions` 内存索引** 是缓存性质的——丢了不要紧，可以从 SQLite 重建。

#### 语义记忆（SemanticMemory）⭐ 最复杂
- **核心**：Neo4j（图）+ Qdrant（向量）协同。
  - Neo4j 存概念节点和它们之间的关系（IS_A、HAS_PART、CREATED_BY 等）；
  - Qdrant 存概念的语义指纹，提供"模糊语言找入口"的能力。
- **写入**：从内容里抽取实体和关系（通常借助 LLM），同步写图和向量库，靠共享 ID 绑定。
- **检索流程**：用向量找入口节点 → 在图上多跳扩展 → 必要时回到向量做二次重排。
- **评分公式**：`(向量×0.7 + 图×0.3) × (0.8 + 重要性×0.4)`
- **`_combine_and_rank_results`** 的关键作用是把"向量榜"和"图榜"合并成统一榜单：用 dict 按 memory_id 去重、缺失的分填 0、按统一公式重排、Top-K 返回。

#### 感知记忆（PerceptualMemory）
- **核心难点**：不同模态的向量维度和语义空间不同，必须**模态分离存储**——文本/图像/音频各开一个 Qdrant 集合。
- **编码器**：text_embedder（文本）+ CLIP（图像）+ CLAP（音频）。
- **跨模态对齐的细节**：要做"用文字搜图"，必须用 CLIP 的文字端把 query 编码，让它和图像向量住在同一个空间；只用普通 text_embedder 是搜不到的——这是教学版的简化。
- **评分公式**：`(向量×0.8 + 时间近因×0.2) × (0.8 + 重要性×0.4)`
- **时间近因**：指数衰减 `exp(-0.1 × age_hours / 24)`，模拟艾宾浩斯遗忘曲线，保底 0.1。

### 5. 评分公式的统一哲学

不同记忆类型的公式形态非常一致：

```
final = (主信号 × w1 + 副信号 × w2) × (0.8 + 重要性 × 0.4)
```

- **主信号**：通常是向量相似度（语义匹配最重要）。
- **副信号**：因类型而异——情景/感知用"时间近因"、工作记忆用"关键词"、语义记忆用"图相似度"。
- **重要性权重**：被严格限制在 [0.8, 1.2]，是"微调"而非"决定性因素"。

> **关键设计哲学**：相关性主导排序、重要性只做 ±20% 微调，避免"重要但不相关"的内容压过"相关但不重要"的内容，保证检索准确性。
> 这些权重不是数学推导出来的，是经验值 + 可调参数，生产环境用评测集做网格搜索调优。

## 三、RAG 系统的总体架构

### 1. RAG 是什么

**Retrieval-Augmented Generation**（检索增强生成）= 在 LLM 生成回答**之前**，先从外部知识库里检索相关文档片段，把它们作为参考资料拼进 prompt，再让 LLM 基于资料作答。

核心目标是 **降低幻觉、引入私有/实时知识、可解释（能溯源）**。

### 2. 工作流程（基础版）

```
原始文档（PDF/Word/HTML/Markdown/CSV/JSON…）
       │
       ▼
①  MarkItDown 转 Markdown：统一格式
       │
       ▼
②  智能分块（chunking）
       │
       ▼
③  Embedding 向量化每块
       │
       ▼
④  入库（Qdrant 等向量库）
       │
       ▼  ← 用户提问时
⑤  query → embedding → 向量库 Top-K
       │
       ▼
⑥  把 Top-K 拼进 prompt 喂 LLM → 回答
```

### 3. RAG 系统设计的关键细节

#### 关键点 A：MarkItDown 统一管道
- 不管原始格式是什么（PDF / Word / Excel / HTML / JSON / CSV…），都先转成 Markdown。
- 好处：
  1. **格式统一**：下游分块/向量化只需要适配一种格式；
  2. **结构保持**：Markdown 自带的标题层级 `#/##/###` 是天然的结构信号；
  3. **可读性**：人也能直接看转换结果，便于调试。

#### 关键点 B：智能分块策略
不是简单按字符切，而是结构感知 + 递归 + 重叠的组合：

1. **结构感知**：按 Markdown 标题层级切，每块带"面包屑路径"作为上下文（如 `[人工智能 > 深度学习 > 神经网络]`）。
2. **大段递归切**：单块超长时，按 `\n\n / \n / 句号 / 空格` 优先级递归二次切。
3. **滑动窗口重叠**：相邻块有 10%~20% 的重叠，防止答案被切口拦腰截断。
4. **预处理**：去掉 Markdown 格式标记、清理代码块、保留语义内容，让 embedding 更聚焦语义。

> 关键直觉：分块的两个矛盾是"块太大语义稀释 vs 块太小上下文不全"，工程上一般 200~800 token + 10~20% 重叠。

#### 关键点 C：高级检索策略
- **混合检索**：向量检索（语义）+ 关键词检索（精确匹配，如人名、产品 ID）；
- **重排（Reranking）**：用更精细的模型对 Top-N 二次打分；
- **MMR 去重**：避免 Top-K 都是同一个意思的不同说法；
- **过滤条件**：可按 metadata（时间、来源、文档类型）做结构化筛选。

#### 关键点 D：检索质量保障
- 设最低分阈值（如 `min_score=0.3`），低于阈值时 prompt 里告诉 LLM"未找到相关资料"，引导回答"资料中未提及"，避免幻觉。
- 把检索结果原文一起返回，方便溯源/审计。

## 四、记忆系统 vs RAG 系统的本质区别

| 维度 | 记忆系统 | RAG 系统 |
|---|---|---|
| **内容来源** | Agent 自己经历/学到的（自动沉淀） | 外部预先准备的文档 |
| **写入方式** | 在线、增量、随时 add | 离线、批量、建库 |
| **可修改** | 是（add/update/forget/consolidate） | 否（一般只读） |
| **存什么** | "我和用户的交互记忆" | "客观知识、私有文档" |
| **典型场景** | 跨会话连贯、个性化 | 客服 FAQ、文档问答、最新资讯 |
| **检索粒度** | 重要性 + 时效性 + 语义 | 主要按语义相似度 |

两者**互补不冲突**——一个 Agent 通常同时配两套：
- 用户问"我之前说我对什么过敏？"→ 记忆；
- 用户问"公司报销额度是多少？"→ RAG。

## 五、本章贯穿全文的几个工程要点

1. **分层与分而治之**：工具层/管理层/类型层/存储层，每层职责单一。不同性质的数据用不同存储引擎（关系/向量/图/文件），通过共享 ID 绑定。
2. **统一接口 + 差异化实现**：所有记忆类型都暴露 `add/retrieve` 等同名接口，但内部公式和存储完全不同。Agent 调用方式始终一致。
3. **评分公式的"主+副+微调"模式**：相关性是主、近因/图相似度等是副、重要性是微调（[0.8, 1.2]）。这种结构在情景、语义、感知记忆里高度复用。
4. **向量空间和模态分离**：不同维度的向量不能混存，CLIP/CLAP 解决跨模态问题的关键是"训练时让两种输入落到同一空间"。
5. **遗忘机制是必须的**：模拟人脑的遗忘曲线（指数衰减）+ 主动遗忘（按重要性/时间/容量），避免无限膨胀和无关信息污染检索。
6. **MarkItDown + 结构感知分块**：把"任意格式 → 高质量 chunk"这条管道工程化，是 RAG 系统能稳定工作的前提。
7. **教学简化 vs 生产差距**：本章很多实现是"够用就好"——TF-IDF 替代 embedding、字符截断替代 LLM 摘要、固定权重替代评测调优、文本编码器和 CLIP 不在同空间等。生产环境需要补齐。

## 六、关键概念速记表

| 概念 | 一句话定义 |
|---|---|
| **MemoryTool** | Agent 操作记忆的统一入口（CRUD + 整合 + 遗忘） |
| **MemoryManager** | 把请求路由到具体记忆类型的协调层 |
| **WorkingMemory** | 内存 + TTL 的临时记忆（短期） |
| **EpisodicMemory** | SQLite + Qdrant 存事件，结构+语义双查 |
| **SemanticMemory** | Neo4j + Qdrant 存知识，图推理 + 向量入口 |
| **PerceptualMemory** | 多模态记忆，分集合存储，CLIP/CLAP 编码 |
| **重要性权重 [0.8, 1.2]** | 排序的微调器，不能颠覆相关性主导 |
| **指数衰减 `exp(-0.1×h/24)`** | 时间近因评分，模拟艾宾浩斯遗忘曲线 |
| **MarkItDown** | 任意格式 → Markdown 的统一转换器 |
| **结构感知分块** | 按 Markdown 标题层级切，每块带面包屑路径 |
| **混合检索** | 向量（语义）+ 关键词（精确） + 重排 |
| **MMR** | Max Marginal Relevance，平衡相关性和多样性，防 Top-K 冗余 |
| **min_score 阈值** | 低于阈值视为未命中，避免胡编乱造 |

## 七、一句话总结全章

> **第八章把"让智能体记得住、查得到"拆成两套互补系统**：
> - **记忆系统**模仿认知科学，分四种记忆类型（工作/情景/语义/感知），用不同存储引擎（内存/SQLite/Qdrant/Neo4j）和统一形态的评分公式（"主信号 + 副信号 × 重要性微调"）实现 Agent 自身经历的沉淀、检索和遗忘；
> - **RAG 系统**通过"任意格式 → MarkItDown → 结构感知分块 → 向量化 → 检索 → 拼 prompt"的管道，让 Agent 能现查私有/外部知识。
> 两者通过统一的工具接口暴露给 Agent，分别解决"我经历过什么"和"世界知识是什么"两个问题。
